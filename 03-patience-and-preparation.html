The Email-Based Git Workflow

This deserves its own section because it's fascinating.

Git was designed for email. Linus Torvalds built it for the Linux kernel, which used (and still uses) mailing lists for patches. The tools are built-in:

# Create patches from your commits
git format-patch origin/main

Send them via email

git send-email *.patch --to=maintainer@project.org

Reviewer applies them locally

git am < patch.mbox

Modern platforms like GitHub made this feel obsolete. But it's not dead - it's alive and thriving in communities that value:

  • Decentralization
  • Offline work
  • Text-based review
  • No proprietary platforms
  • Complete control over tools

Projects on SourceHut require this workflow. The Linux kernel requires it. Many smaller projects accept it.

Why this matters to me: I don't have GitHub access yet. But I have email. I have git. I can create patches. I can contribute this way if I choose to.
